# Rider Atlas / JetMyMoto DESIGN.md

## 1. Overview

Rider Atlas is not a route catalog, travel blog, or booking marketplace. It is the premium visual layer for JetMyMoto's airport-based motorcycle departure system.

The design must communicate one idea instantly:

**Land. Ride. The machine, route, and logistics are already staged.**

Airport pages answer: **Where do I launch from?**  
Mission pages answer: **Why this ride, why this bike, why now?**  
A2A pages answer: **Why this one-way logistics deal is rare and valuable.**

The visual language is cinematic, calm, and operational. It should feel more like an expedition briefing from a private hangar than a tourism website.

## 2. Brand Personality

### Keywords

Premium, cinematic, confident, quiet luxury, expedition-grade, operational, editorial, map-driven, machine-led.

### Avoid

Do not create travel-blog energy. Do not use cheap cards, playful illustrations, cluttered UI, discount-site styling, generic backpacker visuals, or overloaded map markers.

### Core product truth

Rider Atlas sells movement, not just routes.

- Airport = product entry point
- Mission = conversion layer
- Motorcycle = machine match
- A2A = logistics advantage

## 3. Color System

The default experience is warm ivory with black ink and restrained gold. Orange is reserved for decisive commercial action.

### Primary palette

- Ink `#111111`: header, major text, dark sections
- Ivory `#F7F3EA`: main page background
- Warm ivory `#FBF7EF`: softer sections and premium editorial blocks
- Gold `#C9A14A`: signature labels, active navigation, premium badges
- Jet orange `#E57A00`: primary CTA, agent button, A2A deal action
- Muted stone `#6F6658`: body copy, metadata, quiet labels

### Usage rules

Use gold for premium signal, not decoration. Use orange only when the user can take action. Use black generously for confidence and contrast. Keep backgrounds warm, never pure gray unless inside functional UI.

## 4. Typography

### Display type

Large headlines use a high-contrast editorial serif:

`Canela, Cormorant Garamond, Playfair Display, Georgia, serif`

Use this for:

- Hero headlines
- Mission titles
- Route feature titles
- Major editorial statements

Headlines should be large, tight, and cinematic. Use short lines. Prefer phrases like:

- Routes worth landing for.
- The first road out.
- Hidden Europe unlocked.
- Machines matched to this mission.

### UI and body type

Use a clean sans-serif:

`Inter, Manrope, Geist, Helvetica, Arial, sans-serif`

Use this for:

- Navigation
- Stats
- Labels
- Body copy
- Buttons
- Operational instructions

### Labels

Labels are uppercase, small, bold, and widely letter-spaced. They create a premium instrument-panel feeling.

Examples:

- CURATED FROM THIS GATEWAY
- SIGNATURE ROUTE
- ONE-WAY LOGISTICS DEAL
- MACHINE MATCH
- FIRST ROAD OUT

## 5. Layout Principles

### Space

Use generous spacing. Let content breathe. Rider Atlas should feel expensive because it is calm.

### Composition

Prefer strong vertical sections:

1. Cinematic hero
2. Decision strip or stats strip
3. Editorial route block
4. Map / route theater
5. Bike match
6. Logistics CTA

### Max widths

- Page max width: 1520px
- Content max width: 1160px
- Reading width: 820px

Do not stretch body copy across the full screen.

## 6. Navigation

The main navigation should feel like a black hangar header.

- Background: ink `#111111`
- Logo: white with gold emphasis on ATLAS
- Active nav: gold
- CTA: orange rectangular button
- Height: 72px to 88px desktop
- Mobile: simple drawer or compact nav, no clutter

Navigation labels:

- ATLAS
- AIRPORTS
- RIDES
- SHOWROOM
- AGENT

## 7. Buttons and CTAs

### Primary CTA

Use orange fill with white text.

Examples:

- Start this mission
- Match a motorcycle
- Claim one-way deal
- Check availability

### Premium CTA

Use gold pill styling for editorial or soft actions.

Examples:

- View route brief
- Explore airport
- See machines

### CTA copy rules

Use action language. Avoid generic labels like Learn More. The CTA should make the product logic clear.

Good:

- Start this mission
- Claim one-way logistics rate
- Match a motorcycle
- Bring your bike
- Rent locally

Bad:

- Read more
- Discover
- Explore now
- Click here

## 8. Cards

Cards should not feel like cheap UI tiles. They should feel like editorial plates or mission dossiers.

### Premium card

Use for decision blocks and content summaries.

- White or warm ivory background
- 1px soft border
- Radius 24px
- Minimal shadow
- Strong heading
- Short operational copy

### Cinematic card

Use for mission imagery.

- Full-bleed image
- Dark overlay
- White serif title
- Stats pill
- Gold label
- Large radius 24px to 32px
- Slow hover scale or image drift

### Hover behavior

Hover should be subtle:

- Image scale: 1.03
- Overlay darkens slightly
- CTA color shifts to gold or orange
- No bouncy animations

## 9. Icons

Use real SVG icons or a loaded icon component. Never allow raw icon text such as `ARROW_RIGHT_ALT` to appear in the UI.

Recommended icons:

- ArrowRight
- MapPin
- Plane
- Gauge
- Bike / Motorcycle
- Route
- ShieldCheck
- Clock

Icon rules:

- Stroke style preferred
- 18px to 24px default
- Use sparingly
- Pair with labels only when it improves scanning

## 10. Imagery

Imagery must be cinematic and geographically believable.

### Image rules

- Use real roads, mountains, coastlines, passes, lakes, deserts, or machines
- Avoid stock-tourist smiles
- Avoid generic city postcards
- Avoid random landscapes that do not fit the route
- Prioritize mood accuracy if exact location assets are not available

### Required image themes

- alpine_pass
- dolomites
- lake_luxury
- riviera_coast
- tuscany_vineyard
- balkan_mountain
- adriatic_coast
- danube_gorge
- iberia_highland
- atlantic_coast
- motorcycle_showroom

### Motorcycle images

Bike images from Firebase Storage should feel like a curated showroom, not a parts catalog.

Use:

- Clean cutouts or consistent background treatment
- Strong model name
- Fit-for-mission explanation
- Category badges

Example:

**BMW R1250GS**  
Best for long Balkan distance, mixed surfaces, and loaded touring comfort.

## 11. Airport Page System

Airport pages are launch platforms. They are not lists of routes.

Required sections:

1. Hero with cinematic video or image
2. Dual path decision: Bring your bike / Rent locally
3. Journey bridge: land to ride transition
4. Curated missions: routes worth landing for
5. System reveal: staging, logistics, readiness
6. Engine section: logistics or fleet depending on path

### Airport page tone

Operational and emotional. Not poetic fluff.

Good copy:

- Land at Milan Malpensa. Your route is already staged.
- From terminal friction to first road out.
- Selected rides from this gateway, chosen for rhythm, scenery, and the first road out.

Bad copy:

- Discover amazing adventures in beautiful destinations.
- Explore hidden gems and unforgettable memories.

## 12. Mission Landing Page System

Mission pages turn a route into a premium expedition product.

Required sections:

1. Cinematic mission hero
2. Start / end airport and stats strip
3. Mapbox route theater
4. Why this ride exists
5. Day-by-day mission brief
6. Motorcycle match
7. Logistics strip
8. A2A offer box when applicable
9. Final CTA

### Mission hero

Must include:

- Mission title
- Start airport to end airport
- Days / distance / difficulty
- Mission type badge
- Primary CTA

### Round-trip mission tone

Premium, cinematic, destination-led.

Example:

**Val d'Orcia: The Tuscan Dream Run**  
Rome clears behind you. Tuscany opens in long curves, cypress lines, vineyard roads, and old stone towns.

### A2A mission tone

Tactical, rare, commercially sharp.

Example:

**One-Way Deal: Belgrade to Dubrovnik**  
You help move the fleet. We cut the rate. Start inland, finish on the Adriatic, hand over the bike, fly out.

## 13. Mapbox Route Theater

Mapbox is not just a map. It is the route intelligence layer.

Use Mapbox to show:

- Route line
- Start airport pin
- End airport pin
- POI pins
- Day segments
- Terrain mode
- Distance
- Elevation profile when available
- Static map preview for SEO and sharing

### Map styling

- Use restrained labels
- Avoid too many markers
- Highlight the route line clearly
- Use dark overlay or warm map styling depending on page theme
- Do not let the map dominate the entire page unless it is the route theater section

### Copy pattern

Use the phrase **First road out** for the first route explanation.

Example:

From BEG, the route clears the city and bends toward the Danube corridor before the gorge walls close in.

## 14. A2A Design Pattern

A2A missions are the price-killer and logistics engine. They must visually feel different from standard editorial missions.

### A2A badges

Use clear labels:

- ONE-WAY DEAL
- LOGISTICS DISCOUNT
- LIMITED CORRIDOR
- PARTNER REPOSITION

### A2A copy

Use clear commercial explanation:

This mission helps reposition partner fleet across the network. That means limited availability and better pricing than standard rental loops.

### A2A CTA

Use orange CTA:

- Claim one-way deal
- Check one-way availability
- Match this corridor

Do not hide the commercial advantage.

## 15. Bike Match System

Every mission page should include machines matched to the route.

### Section title

**Machines matched to this mission**

### Card content

- Bike image
- Model name
- Category
- Why it fits this mission
- CTA

### Matching logic examples

Alpine / hard / long distance:

- BMW R1250GS
- Honda Africa Twin
- Ducati Multistrada V4
- KTM Adventure

Coastal / luxury / fast touring:

- Ducati Multistrada V4
- BMW R1250RS

Weekend / easy / city escape:

- BMW F750GS
- Yamaha MT-07
- Honda CB650R
- Ducati Scrambler

## 16. Page Themes

### Alpine Technical

Used for Alps, Dolomites, Swiss passes, Grossglockner, Stelvio.

Feel: precise, high altitude, elite, technical.  
Colors: ivory, ink, gold, alpine blue.  
Imagery: passes, hairpins, snowline, lakes, stone hotels.

### Coastal Luxury

Used for Riviera, Amalfi, Croatia coast, Portugal Atlantic.

Feel: sun, sea, slow wealth, polished movement.  
Colors: warm ivory, riviera blue, gold.  
Imagery: coastal roads, cliffs, harbors, golden hour.

### Balkan Raw

Used for Croatia, Bosnia, Serbia, Montenegro.

Feel: hidden Europe, rugged, underpriced, cinematic grit.  
Colors: ivory, ink, balkan green, bronze.  
Imagery: mountains, canyons, coast, border roads, fortress roads.

### Cultural Grand Touring

Used for Tuscany, Loire, Burgundy, castles, vineyards, cities.

Feel: heritage, architecture, vineyards, refined touring.  
Colors: warm ivory, bronze, muted stone, gold.  
Imagery: old towns, vineyards, villas, rolling roads.

### A2A Logistics

Used for one-way reposition missions.

Feel: tactical, limited, high-value, operational.  
Colors: ink, orange, gold, white.  
Imagery: map lines, airport pins, handover moments, road transitions.

## 17. Motion

Motion should feel premium and restrained.

Use:

- Slow image scale on hover
- Smooth section reveal
- Map line draw when entering route theater
- Subtle card lift
- Fade and translate, not bounce

Avoid:

- Springy playful animation
- Excessive parallax
- Spinning icons
- Fast flashing transitions

## 18. Accessibility

- Maintain readable contrast at all times
- Do not place small text directly on bright image areas without overlay
- Buttons need clear focus states
- Interactive cards must be keyboard accessible
- Body copy should be at least 16px
- Avoid all-caps long paragraphs

## 19. Content Rules

### Voice

Write like an expedition operator, not a travel blogger.

Use:

- operational confidence
- destination precision
- short premium statements
- concrete route language

Avoid:

- bucket list
- hidden gems everywhere
- unforgettable memories
- wanderlust
- explore paradise
- generic adventure cliches

### Copy formulas

Airport intro:

`Land at [airport]. The first road out is already selected.`

Mission intro:

`[Start] clears behind you. [Terrain/destination] takes over within hours.`

A2A intro:

`Start at [origin]. Finish at [destination]. You move the fleet; the rate moves with you.`

Bike match intro:

`This route rewards machines built for [terrain], [rhythm], and [distance].`

## 20. Do and Don't

### Do

- Use large editorial serif headlines
- Use warm ivory space
- Use gold sparingly
- Make CTA hierarchy obvious
- Show route stats clearly
- Make A2A offers commercially visible
- Use real geography in titles and copy
- Keep pages calm and premium

### Don't

- Do not make it look like a marketplace grid
- Do not make it look like a travel blog
- Do not use generic tourist phrases
- Do not overuse orange
- Do not use random landscape images
- Do not hide the airport logic
- Do not allow raw icon text to render
- Do not overload maps with pins

## 21. Quality Bar

Every generated screen should pass this test:

Would this page make a rider believe the bike, route, and logistics are already handled before they land?

If yes, it belongs to Rider Atlas.
If no, simplify, sharpen, and make the system visible.
