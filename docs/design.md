# JetMyMoto / RiderAtlas Master Design & UI/UX Guidelines (`design.md`)

## 1. CORE PLATFORM PHILOSOPHY
JetMyMoto is a **Mobility Access Platform** and a **Logistics OS** for motorcycle travel. It is not a standard shipping website or a simple rental listing.
*   **The Dual-Engine UX:** Every key decision page must present a strict **50/50 visual balance** between Logistics (`[ Bring Your Bike ]`) and Rentals (`[ Rent a Bike ]`).
*   **Contextual Commerce:** Products (motorcycles) are never shown in isolation. They are always tied to real-world logistics hubs (Airports) and terrain (Routes/Destinations) via the `GRAPH` engine.

## 2. BRAND DNA & AESTHETIC VIBE
The UI must feel like a **"Luxury Showroom + Tactical Tool"**. 
*   **Inspirations:** Apple Store product grids, Tesla interfaces, and Military Command Center dashboards.
*   **Emotional Goal:** Users are not "browsing" a store; they are planning a mission. The design must project elegance, precision, and immense trust.
*   **Layout Style:** High-end dark aesthetic utilizing glassmorphism, cinematic imagery, and structured, data-heavy side panels.

## 3. COLOR SYSTEM (DARK MODE FIRST)
The platform operates strictly in Dark Mode to highlight cinematic product photography, reduce screen glare, and conserve mobile battery. 
*   **Surface Colors:** **Do NOT use pure black (`#000000`) for backgrounds.** It causes severe eye strain and "halation" (blurry text) for users with astigmatism. Instead, use **Deep Dark Grays** (e.g., `#121212`, `#050606`, or `#574C43`) for main surfaces to maintain visual hierarchy and depth.
*   **Tactical Accents:** Use **Amber, Orange, and Gold** (e.g., `#8B3202`, `#A76330`, `#CDA755`) to highlight Calls-to-Action (CTAs), active filters, and tactical warnings.
*   **Text Contrast:** Use light-colored text at roughly 87% opacity against dark gray backgrounds to achieve optimal, comfortable contrast. 

## 4. TYPOGRAPHY & DATA DISPLAY
Because JetMyMoto functions as a "Command Center" and a "Logistics OS," the typography must handle dense dashboards, grids, and technical specifications effortlessly.
*   **Headers:** Use clean, **bold sans-serif fonts** for strong structural hierarchy, and employ **italic uppercase typography** for tactical headers or system status labels.
*   **Data-Heavy UI Fonts:** For pricing, spec tables, and operational readouts, use dashboard-optimized fonts like **Inter**, **Source Sans 3**, or **IBM Plex Sans**. 
*   **Tabular Numerals (CRITICAL):** All pricing, distances, and metrics must use fonts that support `font-variant-numeric: tabular-nums`. This ensures that numbers in columns align perfectly without manual spacing, giving the platform a precise, technical feel.

## 5. UI COMPONENTS & INTERACTION PRINCIPLES
*   **The Apple-Style Product Card:** Rental bikes should be displayed in a curated grid format. Cards must feature high-quality imagery, clear pricing tags, and a **"Smart Layer"** indicating the bike's optimal terrain (e.g., *Best for: Dolomites*). Hover effects should be used to reveal quick actions like `[ Add to Cart ]` or `[ Request ]`.
*   **Interactive Multiple Filters:** Showrooms must implement dynamic filters (e.g., Type, Price, Brand, Terrain) using React State Hooks or Context API to ensure instant, seamless sorting without page reloads. 
*   **Progressive Disclosure:** To prevent cognitive overload on complex dashboard screens (like `GlobalTower` or `AirportTemplate`), hide secondary information by default. Use drill-downs, accordions, and hover states to reveal details only when the user needs them. 
*   **Visual Hierarchy & Trust:** Prioritize important elements through size and color. Use grayscale-to-color hover states on global partner logos (e.g., DHL, Allianz) to establish a massive "Trust Infrastructure" layer right before the main CTAs.

## 6. FRONTEND ARCHITECTURE CONSTRAINTS
All UX/UI designs must be implemented within the strict rules of the JetMyMoto network engine:
1.  **Single Data Source:** All UI components **must read exclusively from the `GRAPH` object** generated by the Core Data Engine. 
2.  **Stateless Rendering:** Components must function as pure render layers presenting derived data. AI agents must not build complex local states or duplicate data-fetching pipelines.
3.  **Strict Index Consumption:** UI components (like the `RentalGrid` or `AirportTemplate`) are prohibited from manually filtering raw arrays (e.g., iterating over `GRAPH.routes`). They must only consume pre-calculated indexes (e.g., `GRAPH.rentalsByAirport`, `GRAPH.routesByDestination`).
